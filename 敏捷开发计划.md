# 多维人格交叉量表系统 — 敏捷开发计划

## 总览

- **项目周期**：预估 8 个 Sprint（每个 Sprint 2 周，共约 16 周）
- **开发模式**：Scrum
- **团队角色建议**：1 前端 + 1 后端 + 1 产品/设计（或全栈 + 产品）
- **技术栈建议**：
  - 前端：React / Vue 3 + D3.js / ECharts（可视化）
  - 后端：Python（FastAPI / Flask）或 Node.js
  - 数据库：PostgreSQL / MongoDB
  - ML 部分：scikit-learn

---

## Epic 分解

| Epic | 对应模块 | 涉及 Sprint |
|:-----|:---------|:------------|
| **E1 - 项目基建** | 技术选型、项目初始化、数据库建模 | Sprint 1 |
| **E2 - HEXACO 量表引擎** | 量表题库、答题流程、评分逻辑 | Sprint 1-2 |

## 风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|:-----|:-----|:---------|
| 量表题项设计需要心理学专业知识 | 高 | 优先基于现有 HEXACO-100 题项进行改编，后续请专家审核 |
| 机器学习防作弊需要大量训练数据 | 中 | Sprint 7 前先实现规则基线版（RT 阈值检测），ML 版作为优化 |
| 心灵剧场 UI 交互复杂度高 | 中 | Sprint 6 先实现静态布局 + 切换交互，动画细节后期迭代 |
| 常模数据不足，T 分数偏差大 | 中 | 初始使用预设参数，上线后积累用户数据逐步校准 |

---

## 开发优先级建议

如果资源有限，建议按以下优先级推进：

1. **Sprint 1-3（MVP 核心）** — 完成 HEXACO 量表 + 第一层展示，这是整个系统的基础
2. **Sprint 4（MBTI 映射）** — 降维映射是用户最关心的转化层
3. **Sprint 6（心灵剧场）** — 核心竞争力，差异化亮点
4. **Sprint 5（交叉验证）** — 增强信效度
5. **Sprint 7（防作弊）** — 锦上添花，可后续迭代
