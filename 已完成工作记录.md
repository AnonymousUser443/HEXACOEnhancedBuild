## 一、设计阶段

### 文件：`多维人格交叉量表与网页展示系统设计.md`

一份完整的设计报告，涵盖：

  - 九型人格：核心动机与恐惧辨析
- **防作弊机制**：情境判断测验（SJT）+ 响应时间（RT）采集 + 机器学习异常检测
- **降维映射算法**：逻辑斯蒂回归，HEXACO → MBTI 四维度连续概率光谱
- **三层前端展示**：特质雷达图 → MBTI 光谱 + 九型标签 → 心灵剧场交互 UI

## 三、Sprint 1 — 项目基建与 HEXACO 数据建模（已完成）

### 技术栈

| 层级 | 技术选型 |
|:-----|:---------|
| 前端 | Vue 3 + Vite + Vue Router + Pinia + ECharts |
| 后端 | Node.js + Express |
| 数据库 | SQLite（sql.js 纯 JS 实现，避免原生编译问题） |
| API 风格 | RESTful JSON |

### 项目结构

```
backend/
├── package.json
├── src/
│   ├── index.js              # Express 服务器入口（异步启动）
│   ├── database.js           # sql.js 初始化 + 建表
│   ├── dbhelper.js           # run()/get()/all() 查询辅助
│   ├── seed.js               # 种子数据
│   └── routes/
│       ├── questions.js      # 维度/子面/题项查询 API
│       └── sessions.js       # 会话管理 + 答题 + 评分 API
└── data/
    └── personality.db        # SQLite 数据库文件

frontend/
├── package.json
├── vite.config.js            # Vite 配置（含 API 代理 :5173 → :3000）
├── index.html
└── src/
    ├── main.js               # Vue 入口
    ├── App.vue               # 根组件
    ├── router/index.js       # 路由配置
    └── views/Home.vue        # 首页
```

### 数据库模型

```
dimensions        → 6 条（H/E/X/A/C/O）
facets            → 24 条（每维度 4 子面）
questions         → 96 题（每子面 4 题，部分反向计分）
users             → UUID 主键，匿名注册
sessions          → 会话状态（in_progress / completed）
answers           → 答案记录（含 response_time_ms）
```

### 完整 API 清单

| 方法 | 路径 | 功能 |
|:-----|:-----|:-----|
| GET | `/api/health` | 健康检查 |
| GET | `/api/dimensions` | 获取所有维度 |
| GET | `/api/dimensions/:code/facets` | 获取某维度的子面 |
| GET | `/api/questions` | 获取所有题目（含维度/子面） |
| POST | `/api/sessions` | 创建新会话（匿名） |
| POST | `/api/sessions/:id/answers` | 提交答案（可覆盖） |
| GET | `/api/sessions/:id/progress` | 查询答题进度 |
| POST | `/api/sessions/:id/complete` | 完成会话 |
| GET | `/api/sessions/:id/result` | **HEXACO 评分结果** |

### 评分引擎逻辑

- 反向计分：`reverse_scored ? 6 - answer_value : answer_value`
- 子面得分：子面内题项原始分均值
- 维度得分：子面均值的均值
- 百分位映射：`((avg - 1) / 4) × 100`（1-5 Likert → 0-100）
